Exercise 6.2
Getting familiar with the Software Renderer Library (SRL)
  • First thing: notice that we are still using opengl to display the image produced using SRL in the screen (i.e. communicate with the video card) and glfw for input and window management.
  • srl_frame_buffer is where we store images produced with our renderer.
  • The srl_renderer class controls the rendering pipeline and implement the generic parts of it. Notice that there are three specializations of that class, one per primitive type (srl_point_renderer, srl_line_renderer and srl_triangle_renderer), where the primitive specific functions are implemented. We use polymorphism to be able to swap the renderer that we use.
  • srl_types has the definition of the special object types we use in our renderer, such as the vertex, fragment, color and primitives.
  • srl_point_renderer and srl_line_renderer are the only renderers that are completelly implemented. Go through the srl_renderer and srl_point_renderer member functions and comments to get familiar with the steps of the rendering pipeline. We will discuss in class (week 7) how these functions fit in the graphics pipeline.
  • In the current implementation you can get to render points and lines, and triangles with some limitations (no interpolation or depth testing). It should produce images like the ones in the left (use keys 1, 2 and 3 to switch the rendering mode to points, lines and triangles).
