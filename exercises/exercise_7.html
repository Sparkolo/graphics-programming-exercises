Exercise 7 - Clipping and culling

Learning objectives
  • Implement a triangle clipping algorithm.
  • Implement a triangle culling algorithm.
  • Understand how clipping and culling affect render performance.
Exercise 7.1
 Triangle clipping algorithm
  • Follow the algorithm in the slides to implement the triangle clipping algorithm agains the frustum volume defined by w >= x,y,z >= -w.
  • Noticed that:
    • a scale is being applied to the vertex positions after clipping, this is to help visualizing if the clipping is correct;
    • the line clipping algorithm in srl::LineRasterizer may be helpful;
    • after clipping, your program should produce images like you see in the bottom-left.
Exercise 7.2

Back face culling gone wrong 		Back-face culling algorithm
  • Reject triangle primitives when they are not facing the camera (follow the instructions in the slides).
  • The application should run around 2 times faster after implementing the back face culling algorithm.
Exercise 7.3
Debugging and fixing your implementation
  • If you produced teared images (left), then the problem is in the winding order of the new triangles produced during clipping.
  • The vertices of new triangles have to be added in the right order (counter-clockwise), otherwise they will be erroneously flagged by the back-face culling algorithm.
    • If you see the inside of the airplane, then the winding order test is not correct (z direction of the normal).
    • If you see the inside of only some of the airplane parts, then the winding order of these parts need to be redefined (e.g. the vertices composing the mesh).
